# DeskHalloumi 0.2.0

Released July 18, 2026.

DeskHalloumi 0.2.0 is the first release under the new project name. It turns the
former `unilii` workspace into a cohesive desktop-control toolkit while keeping
legacy command, environment, configuration-path, and systemd compatibility for
the pre-1.0 transition.

## Highlights

- **Two safe i3/X11 shortcut paths.** Standard press/release shortcuts can be
  exported to an atomic i3 include, while the selective native X11 backend
  supports hold, modifier-release, repeat, cooldown, priority, and consume
  semantics.
- **Migration-aware sxhkd import.** The importer handles releases, paired
  sequences, same-class numeric/alphabetic ranges, optional empty elements, and
  escaped braces. Unsupported or approximate semantics are reported rather than
  silently activated.
- **Active i3 conflict auditing.** Recursive includes, variables, modes,
  `bindsym`, and `bindcode` entries are inspected with source-located conflict
  reports.
- **Dynamic keyboard lifecycle.** Evdev observation adds newly connected
  keyboards, retires removed devices independently, and suppresses stale events
  when an event path is reused.
- **Cross-process desktop actions.** A private, versioned Unix-socket protocol
  connects standalone hotkeys with bar, tray, widget, and managed-menu actions.
- **Improved popup UX.** CopyQ, filter-tab, and i3 tree frontends have consistent
  keyboard navigation, managed single-instance behavior, and clearer status or
  shortcut guidance.
- **Release safety.** CI runs formatting, the live-session command audit, the
  full safe test suite, isolated Xvfb+i3 integration, strict Clippy, tag
  validation, and deterministic artifact packaging.

## Compatibility

Primary commands use the `deskhalloumi-*` prefix. Lightweight `unilii-*`
launchers remain available. New configuration and runtime paths use
`~/.config/deskhalloumi` and `$XDG_RUNTIME_DIR/deskhalloumi`; legacy locations
remain readable without destructive migration.

Legacy desktop application IDs remain unchanged in 0.2.0 so existing i3 window
rules continue to match.

## Supported boundary

i3/X11 is the tested global-hotkey environment. Iced rendering may operate on
Wayland, but Sway/Wayland global-hotkey, i3 IPC, `xrandr`, and `xset` parity is
not claimed.
