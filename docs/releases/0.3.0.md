# DeskHalloumi 0.3.0

Released July 19, 2026.

DeskHalloumi 0.3.0 makes the interactive desktop runtime explicitly owned,
bounded, observable, and testable. It consolidates command execution, provider
state, menus, quick-select, and typed actions without removing the compatibility
surfaces introduced in 0.2.0.

## Downloads

## Highlights

- **Owned asynchronous runtime.** Long-lived workers are supervised through
  retained task sets, cancellation tokens, bounded spawn queues, and bounded
  graceful shutdown instead of detached background tasks.
- **One subprocess policy.** UI-sensitive commands use `tokio::process` with
  timeouts, bounded retained output, truncation metadata, working-directory and
  environment support, and Unix descendant-process termination.
- **Generation-safe providers.** Clock, battery, network, audio, system, and
  Tmux use typed startup/loading/fresh/stale/error/disabled/shutdown states.
  Late refreshes and queued messages from replaced provider instances cannot
  overwrite newer data, while failures retain the last known good value.
- **Bounded provider pressure.** Refresh concurrency is globally bounded,
  duplicate in-flight refreshes coalesce, and dropped or overwritten updates
  are counted.
- **Renderer-neutral interaction contracts.** Tray, widget, custom launcher,
  filter-tab, and system menus share a typed menu model with standardized
  loading, busy, stale, disabled, and error states.
- **One-shot quick-select.** Ordered home-row-first shortcuts are rendered as an
  overlay; a mapped key invokes exactly one action, while any other key aborts
  and is consumed.
- **Introspection and diagnostics.** The CLI can list modules, menus, actions,
  and hotkeys, invoke typed actions, and query live runtime metrics over the
  private action bus.
- **Visible failures.** Bounded action history records running, successful,
  failed, timed-out, and cancelled operations with duration and failure detail.

## Useful diagnostics

```sh
deskhalloumi list-modules --json
deskhalloumi list-menus --json
deskhalloumi list-actions --json
deskhalloumi list-hotkeys --json
deskhalloumi runtime-metrics --json
deskhalloumi-bar --runtime-contract
```

The separate `deskhalloumi-bar` command remains intentionally synchronous and
headless. It is a configuration, fixture, scheduler, reload, and text-render
reference runtime—not a second production panel daemon.

## Compatibility and migration

No configuration migration is required from 0.2.0. Primary commands continue
to use the `deskhalloumi-*` prefix, while lightweight `unilii-*` launchers,
legacy configuration/runtime paths, and legacy desktop application IDs remain
available during the pre-1.0 transition.

The action-bus protocol remains private, versioned, user-scoped, size-bounded,
and timeout-bounded. The new runtime-metrics request is additive.

## Supported boundary

i3/X11 remains the tested global-hotkey environment. Iced rendering may operate
on Wayland, but Sway/Wayland global-hotkey, i3 IPC, `xrandr`, and `xset` parity
is not claimed.

The published archive targets `x86_64-unknown-linux-gnu`; it is not a claim of
Windows, macOS, musl, or Linux AArch64 support.
